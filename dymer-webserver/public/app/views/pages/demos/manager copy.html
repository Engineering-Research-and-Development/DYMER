Following the code needed to view a light version of DYMER administration system.
With the light version it is possible:

  • Upload and View a Model
  • Upload and View a Template
  • Create new Entity
Come si può notare dal codice presente nel file kms_manager.html, il tutto può essere realizzato importando solamente un singolo file (‘kms.form.js’) ed inserendo nella function mainKmsForm() i metodi da invocare.

Kms.form.js si occuperà di importare ed eseguire tutti files necessari per il funzionamento della sezione di amministrazione.

La function mainKmsForm() è la funzione che viene invocata al termine del caricamento di tutti i files.
Nella mainKmsForm() viene utilizzata la funzione ‘loadFormlist(sourceUrl, target, datapost)’ che necessita del passaggio di tre parametri:
  • sourceUrl : url del microservizio da dove recuperare i contenuti (l’indirizzo è mappato in un json di confugurazione e quindi basta utilizzare la funzione getendpoint e passare il microservizio che vogliamo interrogare form/template/entity)
  • target : l’elemento dom dove renderizzare la lista
  • datapost : la query da applicare alla chiamata

Creazione Form

Gli elementi presenti nel form che dovranno essere serializzati per poter creare un'entità in elastichsearch, dovranno avere l'attributo name valorizzato secondo il costrutto data['key'].
Il form html dovrà contenere necessariamente i seguenti elementi: 

      <input name="instance[index]" type="hidden" value="index-entità">
      <input name="instance[type]"  type="hidden" value="type-entità">
instance[index] e instance[type] possono essere rapportati al 'database' e 'tabella' dei classici Database RDBMS.
