Redis Cache
  # Introduction   # Setup   # Enable/Disable

Introduction

Entities are stored in the Elastic Search Database; the Entities microservice generates a specific query to retrieve them. As the number of entities stored increases, generating several queries would be an onerous operation, so a cache could improve the performance. The idea is to store the query and the response to avoid executing a previously performed query. If a new query is already stored in the cache, the Redis Module will provide its relative response bypassing the query execution. Of course, different users can obtain different responses according to their own profiles and permissions, so the cache has to store some additional information, like the origin or the user identification.

Setup

To use the Redis module, you need to enable it from the entities microservice configuration file (\dymer-entities\config\config.json in the json object services.entity). Set the following section: 

"cache": {
    "protocol": "http",
    "host": "127.0.0.1",
    "ip": "cache",
    "port": <redis port>,
    "user": "redis",
    "password": "<redis password>",
    "isEnabled": true
    }
When the microservice turns on, an initialization function is called.

Enable/Disable

You can enable or disable redis cache from GUI in DASHBOARD > General Settings section In Redis section you can enable or disable it via checkbox
